Entity Object

The Name Parser generates an entity object.

{ type: 0,
name: "",
dba:"",
members: [ ],
_ID: "" }

Type is a number:

  • 0 = individual
  • 1 = partnership (married couples)
  • 2 = corporation or company

Name is the entity name in default format. For individuals this will be last name, first name.
For partnerships it will be last name, first name & first name #2.
For companies it will be exactly what was typed in.

Dba is the name the entity does business as. This is optional.

Members is a collection of the names associated with the entity. For individuals and companies there will be only 1 element. For partnerships it will have two. The member element, or name object, holds the various parts of the name.

_ID is assigned a uuid if there is none already.

Parts of a name may be identified explicitly.

Char Description
= the entity is defined. ie. "=i" for individual; "=2" for corporation
"=" (no other chars) to open the subform for manual entry
$ at start of string indicates a company name. "$Walt Disney"
& at start of string indicates a partnership name.
dba indicates the following string is 'doing business as' name.
[...] explicitly defines the dba name. [My Great Company]

The following only apply to individuals:
< indicates the following word is a prefix.
! indicates the following word in the name is the preferred name.
~ indicates the following word is the preferred name and not part of the usual name.
" ... " words inclosed in double quotes are treated as a single word.
eg. Klaus "von Barlow"; first name = 'Klaus', last name = 'von Barlow'
(...) words enclosed in parens are alternate spelling of the first or last name.
/.../ words enclosed in / characters are the last name.
The position within the string doesn't matter and commas are ignored.
, It the last name isn't defined the words precedding a comma become the last name.
